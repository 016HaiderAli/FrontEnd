Coding/Development


When creating the application, we followed the design and defined specs as we built the actual code. The application was created using the design as a guide and a variety of tools, including compilers (such as Android Studio and Visual Studio Code), a programming language (Dart), an open-source software development toolkit (Flutter), various APIs (such as Google Map and Payment Method), and Firebase for various functionalities, such as login validation, phone validation, and email validation.

Testing


In order to ensure that the end-user experience would not be harmed, an application was tested once it was fully constructed. Building software was not the end of the process. To give users a bug-free and error-free experience, we combed over the software repeatedly during this stage, trying to detect, note, and solve errors as much as possible.

Deployment & Evaluation.


The system is made accessible to the user. When using the system, users comment on whether it can be made better or if anything needs to be changed. Improvements are made depending on user input to ensure that the system fully satisfies the criteria.

FUNCTIONAL & NON-FUNCTIONAL REQUIREMENT:
Functional Requirements
• Chat system: Most on-demand tutor apps develop an in-app chat messenger to allow students to communicate with tutors and vice-versa. Students can ask questions about the fee structure, class timings, and the learning process.
• Cross-platform applications: Applications will be available on various platforms (Android, IOS, Web, and macOS).
• Rate & Review: Users will be able to rate the experience they had with the tutors. The tutor profile includes information about their experience, expertise on a particular subject, and more. The section also comprises of reviews and ratings given by other students.
• Multiple payment options: Users will have various methods for payment transactions, that is, via bank transfer, cards, and online payment methods.
• Feedback form to improve the app: Almost all the on-demand tutor applications can submit ratings and reviews about the tutor. The feedback submitted by students helps other students to know about the tutor.
• Social media integration: Users can register quickly on an application using social media such as Google and Facebook.

Non-Functional Requirements
• Performance Requirements: Fast, efficient, and responsive. It works on the most significant number of devices. The content servers will be available 24/7 with a high response time.
• Safety Requirements: No irrelevant or outdated content can cause things in the tech sector to change frequently, and old information can lead to a waste of time for the user in the future. Security Requirements: The app will use encryption in the database to store user details with confidentiality and integrity. So, the users can trust us while providing their details.
• Software Quality Attributes: It is a mobile application that is easy to access anywhere.
